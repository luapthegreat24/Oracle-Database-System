Pet Care Scheduling Database System (Oracle 19c)
Overview

This project is a database management system developed using Oracle 19c, designed to manage and streamline pet care scheduling. It maintains structured records of customers (pet owners), their pets, and corresponding treatment schedules to ensure efficient and reliable data handling.

The system follows a modular architecture, where components are interconnected through a navigation structure, allowing seamless interaction between related data and functionalities.

Features
Customer Management
Stores and manages pet owner information.
Pet Records Management
Maintains detailed records of pets, linked to their respective owners.
Treatment Scheduling
Organizes and tracks treatment schedules for pets.
CRUD Operations
Supports Create, Read, Update, and Delete operations for:
Customer information
Pet records
Treatment schedules
Modular Navigation System
Implements interconnected modules that allow efficient navigation and workflow across different system components.
System Architecture

The system is based on a relational database structure consisting of interconnected tables that define relationships between:

Customers (Owners)
Pets
Treatment Schedules

Primary and foreign keys are used to enforce relationships, ensuring:

Data consistency
Referential integrity
Efficient querying and updates
Technologies Used
Database: Oracle 19c
Query Language: SQL
Design Approach: Relational Database Design
Core Modules
Customer Module
Manages owner information and records
Pet Module

Treatment Module
Handles scheduling and treatment records
Navigation System
Connects modules to enable seamless user interaction

Veterinary clinics
